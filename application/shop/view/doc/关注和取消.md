
## 关注接口

功能：用户a（发起用户）关注用户b（目标用户）。会导致b的粉丝数加1.
~~~
/user/user/focus
~~~
~~~
POST
~~~

| 请求参数字段        | 请求参数含义  |
| -------- |:------|
|self_uid|  发起用户id|
|focus_uid| 被关注用户id  |

| 返回字段        | 类型 |含义  |
| -------- |:------|:------|
| 无       |   |

| 错误        | 
| -------- |
| 1、已经关注，用户不存在等 | 


## 取消关注接口

功能：用户a（发起用户）取消关注用户b（目标用户）
~~~
/user/user/un_focus
~~~
~~~
POST
~~~

| 请求参数字段        | 请求参数含义  |
| -------- |:------|
|self_uid|  发起用户id|
|focus_uid| 被取消关注用户id  |

| 返回字段        | 类型 |含义  |
| -------- |:------|:------|
| 无       |   |

| 错误        | 
| -------- |
| 1、原来就没有关注，用户不存在等 |
