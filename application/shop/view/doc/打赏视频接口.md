
可能服务端返回错误（例如波币不够），格式如下，没有data字段了。

    [
        "message":"错误信息",
        "code":0, //0表示错误。
    ]

所以，客户端必须判断code为1，是正确。

说明：**以下所有接口的请求参数，并没有顺序要求，可以随意颠倒，都是get请求。**

----
## 打赏视频接口

功能：打赏视频
~~~
/shop/dashang/index
~~~
| 请求参数字段        | 请求参数含义  |
| -------- |:------|
|uid|  打赏人用户id，是付钱的人|
|room_id| 短视频或直播，或回播的room_id，不是主键id  |
|type| 1-7之间的整数，参加 《打赏视频价格表》  |

| 返回字段        | 类型 |含义  |
| -------- |:------|:------|
| current_gold       | int | 付钱后，当前用户波币数 |


| 错误代码        |含义  |
| -------- |:------|
| -202      |波币余额不足  |
